---
title: "LoRA fine-tuning"
description: "Fine-tune VibeVoice-ASR on your own data using Low-Rank Adaptation (LoRA) for domain adaptation, hotword bias, and accent coverage."
---

LoRA fine-tuning lets you adapt VibeVoice-ASR to domain-specific vocabulary, accents, or speaking styles without modifying the full 7B parameter model. The training script wraps HuggingFace `Trainer` with PEFT, so all standard `TrainingArguments` are available alongside VibeVoice-specific options.

<Note>
  The `toy_dataset/` bundled in the repository uses **synthetic audio generated by VibeVoice-TTS** and is provided for smoke-testing only. Use real audio recordings with accurate transcriptions for production fine-tuning.
</Note>

## Prerequisites

<Steps>
  <Step title="Install VibeVoice">
    Install the base package from the repository root:

    ```bash
    pip install -e .
    ```
  </Step>

  <Step title="Install PEFT">
    Install the Parameter-Efficient Fine-Tuning library:

    ```bash
    pip install peft
    ```
  </Step>

  <Step title="Prepare your dataset">
    Organize audio files and JSON label files in a flat directory. Each audio file must have a matching JSON file with the same base name.
  </Step>
</Steps>

## Dataset format

Training data is a flat directory of audio/JSON pairs:

```
dataset/
├── 0.mp3
├── 0.json
├── 1.mp3
├── 1.json
└── ...
```

Each JSON file describes one audio file:

```json
{
  "audio_duration": 351.73,
  "audio_path": "0.mp3",
  "segments": [
    {
      "speaker": 0,
      "text": "Hey everyone, welcome back...",
      "start": 0.0,
      "end": 38.68
    },
    {
      "speaker": 1,
      "text": "Thanks for having me...",
      "start": 38.75,
      "end": 77.88
    }
  ],
  "customized_context": ["Tea Brew", "Aiden Host", "The property is near Meter Street."]
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `audio_duration` | Yes | Total duration of the audio file in seconds |
| `audio_path` | Yes | Filename of the audio file (relative to the JSON file) |
| `segments` | Yes | List of transcription segments with speaker, text, and timing |
| `segments[].speaker` | Yes | Integer speaker ID |
| `segments[].text` | Yes | Transcript text for the segment |
| `segments[].start` | Yes | Segment start time in seconds |
| `segments[].end` | Yes | Segment end time in seconds |
| `customized_context` | No | Domain-specific terms or context sentences for the `--use_customized_context` feature |

## Training

<Tabs>
  <Tab title="Single GPU">
    ```bash
    torchrun --nproc_per_node=1 lora_finetune.py \
        --model_path microsoft/VibeVoice-ASR \
        --data_dir ./toy_dataset \
        --output_dir ./output \
        --num_train_epochs 3 \
        --per_device_train_batch_size 1 \
        --learning_rate 1e-4 \
        --bf16 \
        --report_to none
    ```
  </Tab>

  <Tab title="Multi-GPU">
    Select the GPUs to use with `CUDA_VISIBLE_DEVICES` and set `--nproc_per_node` to match:

    ```bash
    CUDA_VISIBLE_DEVICES=0,1,2,3 torchrun --nproc_per_node=4 lora_finetune.py \
        --model_path microsoft/VibeVoice-ASR \
        --data_dir ./toy_dataset \
        --output_dir ./output \
        --num_train_epochs 3 \
        --per_device_train_batch_size 1 \
        --learning_rate 1e-4 \
        --bf16 \
        --report_to none
    ```
  </Tab>

  <Tab title="Full options example">
    ```bash
    torchrun --nproc_per_node=4 lora_finetune.py \
        --model_path microsoft/VibeVoice-ASR \
        --data_dir ./toy_dataset \
        --output_dir ./output \
        --lora_r 16 \
        --lora_alpha 32 \
        --lora_dropout 0.05 \
        --num_train_epochs 3 \
        --per_device_train_batch_size 1 \
        --gradient_accumulation_steps 4 \
        --learning_rate 1e-4 \
        --warmup_ratio 0.1 \
        --weight_decay 0.01 \
        --max_grad_norm 1.0 \
        --logging_steps 10 \
        --save_steps 100 \
        --gradient_checkpointing \
        --bf16 \
        --report_to none
    ```
  </Tab>
</Tabs>

## Training options

The script accepts VibeVoice-specific arguments alongside all standard HuggingFace `TrainingArguments`.

### LoRA arguments

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--lora_r` | `16` | LoRA rank — lower reduces trainable parameters, higher increases expressiveness |
| `--lora_alpha` | `32` | LoRA scaling factor — typically set to 2× the rank |
| `--lora_dropout` | `0.05` | Dropout applied to LoRA layers |

### Data arguments

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--data_dir` | `./toy_dataset` | Directory containing paired audio and JSON label files |
| `--max_audio_length` | None | Skip audio files longer than this value (seconds) — useful to exclude outliers |
| `--use_customized_context` | `True` | Inject the `customized_context` list from each JSON file as additional model context |

### Key training arguments

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--per_device_train_batch_size` | `8` | Batch size per GPU |
| `--gradient_accumulation_steps` | `1` | Effective batch size = `batch_size × grad_accum` |
| `--learning_rate` | `5e-5` | Learning rate — `1e-4` to `2e-4` is typical for LoRA |
| `--gradient_checkpointing` | `False` | Recompute activations on the backward pass to reduce GPU memory usage |

<Tip>
  If you run out of GPU memory, enable `--gradient_checkpointing` and reduce `--per_device_train_batch_size` while increasing `--gradient_accumulation_steps` to preserve the effective batch size.
</Tip>
