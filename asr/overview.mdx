## What is VibeVoice-ASR?

**VibeVoice-ASR** is a unified speech-to-text model (7B parameters) designed to handle **60-minute long-form audio in a single pass**. Rather than slicing audio into short chunks like most conventional ASR systems, it encodes the entire recording within a 64K-token context window, preserving global speaker tracking and semantic coherence across the full hour.

The model jointly performs transcription, speaker diarization, and timestamping in one forward pass, producing a structured JSON output that answers **Who** spoke, **When** they spoke, and **What** they said.

## Architecture overview

VibeVoice-ASR is built on a multimodal LLM backbone derived from **Qwen2.5-7B**. Speech is encoded through two parallel tokenizers:

- **Acoustic tokenizer** — captures low-level audio fidelity at 7.5 Hz frame rate
- **Semantic tokenizer** — captures high-level linguistic content

The outputs of both tokenizers are projected into the language model's embedding space via lightweight `SpeechConnector` modules. The language model then autoregressively generates structured JSON transcriptions directly, with no separate decoding stage.

The model supports Flash Attention 2, SDPA, and eager attention backends, and loads in `bfloat16` by default.

## Output format

The model produces a JSON array where each element represents one speaker turn:

```json
[
  {
    "speaker_id": 0,
    "start_time": 0.0,
    "end_time": 4.32,
    "text": "Welcome everyone to today's meeting."
  },
  {
    "speaker_id": 1,
    "start_time": 4.85,
    "end_time": 9.10,
    "text": "Thanks for having me. Let's get started."
  }
]
```

## Language distribution

VibeVoice-ASR supports over 50 languages without any explicit language flag. The training data distribution covers major world languages including English, Chinese, French, German, Italian, Japanese, Korean, Portuguese, Russian, Spanish, Thai, Vietnamese, and many more.

Code-switching — mixing two languages within a single utterance — is handled natively. The model detects and transcribes language transitions without any configuration.

<Note>
  You do not need to specify a language before transcribing. The model auto-detects the language from the audio signal.
</Note>
